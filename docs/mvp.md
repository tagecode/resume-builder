# MVP 功能范围

本文档从 [`docs/prd.md`](./prd.md) 提取 **MVP（最小可行产品）** 必须交付的能力，作为后续开发的验收清单。优先级以 PRD **§6 功能需求** 中的 **P0** 为准，并与 **PRD §9 MVP** 描述对齐。

| 属性 | 内容 |
|------|------|
| 对应 PRD | `docs/prd.md` v1.0 |
| MVP 原则 | P0 全覆盖；不依赖网络；可完成「新建 → 编辑 → 预览 → 导出 PDF」闭环 |

---

## 1. MVP 目标

用户在 **完全离线** 环境下能够：

1. 创建并管理多份简历（空白或基于模板）；  
2. 按 **分区结构** 编辑简历内容，并控制各区块 **显示/隐藏**；  
3. 在 **分栏界面** 中 **实时预览** 版式；  
4. 在至少 **2～3 套** 内置模板间切换且 **不丢字段**；  
5. **自动保存** 与 **手动保存**（含切换简历时的未保存提示）；  
6. 将简历 **导出为 PDF**（纸张与边距等基础可配置）；  
7. 数据 **仅保存在本机**（无需注册）。

## 2. 功能清单（按模块）

### 2.1 简历项目管理

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| PM-01 | 新建简历：空白 / 从模板创建 | 创建后进入编辑器；模板创建时可带预填示例结构 |
| PM-02 | 简历列表：重命名、复制、删除 | 删除需确认；名称合法校验 |
| PM-04 | 多简历切换 | 切换前有未保存变更时：**保存 / 放弃 / 取消** |
| PM-05 | 自动保存与手动保存 | 防抖或定时自动保存；**Ctrl/Cmd+S** 手动保存有明确反馈 |

### 2.2 智能内容编辑

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| CE-01 | 分区编辑 | **个人信息**；**总结/求职意向**；**工作经历**（公司、职位、时间、描述，条目可增减）；**教育背景**；**技能**；**项目经验**；**证书与奖项**；**语言能力**；**自定义模块**（用户可增删自定义区块） |
| CE-04 | 模块显示/隐藏 | 隐藏模块 **不出现在预览与 PDF 导出** 中 |

> **说明**：MVP 阶段 **不要求** 轻量富文本（CE-02，P1）、区块拖拽排序（CE-03，P1）。字段以纯文本或简单多行文本为主即可，避免阻塞 PDF 与分页策略。

### 2.3 模板与样式

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| TS-01 | 内置多套专业模板 | **至少 2～3 套**；分类可简化为标签，后续再扩展 |
| TS-02 | 模板切换与实时预览 | 切换模板 **不丢字段**；无法映射时须有提示或映射策略（MVP 可优先「全模板字段一致」降低复杂度） |

> **说明**：全局样式细调（TS-03，P1）、模块样式微调（TS-04，P2）、用户自定义模板（TS-05，P2）**不纳入 MVP**。

### 2.4 实时预览

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| PV-01 | 分栏：左侧编辑、右侧预览 | 在约定最小窗口宽度内可用（具体数值由 UI 稿/实现确定） |

> **说明**：单页/双页预览模式、分页溢出提示、预览缩放、预览主题等（PV-02～PV-05）**不纳入 MVP**，列入 V1.1 / V1.2。

### 2.5 导入与导出

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| IO-01 | 导出 PDF | 支持 **A4 / Letter**、**边距**、**质量** 等基础选项；中文与常用符号无乱码 |

> **说明**：打印、图片、Markdown、JSON 备份/导入（IO-03～IO-06）**不纳入 MVP**，与 PRD **§9 V1.1** 对齐。

### 2.6 本地数据

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| DS-01 | 完全本地存储 | **无需账号**；除显式联网能力外 **不依赖网络** |

> **说明**：自定义数据目录、版本历史、云同步等 **不纳入 MVP**。

### 2.7 桌面端（Electron）

| ID | 需求 | 验收要点（摘自 PRD） |
|----|------|----------------------|
| DT-07 | 离线可用 | MVP 主流程 **零网络依赖** |
| DT-08 | 跨平台一致 | **Windows / macOS / Linux** 均可完成上述闭环；路径差异在实现层处理 |

> **说明**：系统托盘、全局快捷键、窗口记忆、原生菜单、自动更新（DT-01～DT-06）**不纳入 MVP**，列入后续版本。

## 3. MVP 非功能要求（摘自 PRD §7，裁剪）

| 类别 | MVP 必须满足 |
|------|----------------|
| 性能 | 自动保存使用防抖；编辑时界面可交互 |
| 可靠性 | 持久化写入失败有提示；优先 **原子写** 或临时文件替换，避免半写坏文件 |
| 安全 | 维持 `contextIsolation` + preload **白名单 API**；不向渲染进程暴露任意 Node |
| 隐私 | 简历数据默认仅存本地；无账号体系 |

## 4. 数据模型（MVP 最低要求）

与 PRD **§8** 一致，建议每份简历至少包含：

- `schemaVersion`（便于后续迁移）  
- `resumeId`、`name`、`updatedAt`  
- `sections` / 分区内容（覆盖 CE-01 所列类型）  
- `templateId`  
- 各模块 `visible` 或与 CE-04 等价的显隐配置  

存储形态（单文件 JSON、目录结构或 SQLite）由实现选定，需在开发文档中固定一种 **默认路径**（可对接 PRD 开放问题「默认数据目录」）。

**本仓库实现（Electron 默认）**

- 每份简历：`{userData}/resumes/{resumeId}.json`  
- `userData`：Electron [`app.getPath('userData')`](https://www.electronjs.org/docs/latest/api/app#appgetpathname)，因操作系统与用户而异（例如 macOS 常为 `~/Library/Application Support/Resume Builder/`）。  
- 主题等轻量设置：`electron-store` 写入同一 `userData` 命名空间（与简历 JSON 目录并列，非 `resumes/` 子目录内）。

## 5. MVP 显式不包含（防止范围蔓延）

以下能力 **不在 MVP 交付范围内**，详见 PRD 各条 **P1～P3** 及 **§9** 后续阶段：

- 列表搜索/排序（PM-03）、最近列表与异常草稿恢复（PM-06）  
- 富文本（CE-02）、区块拖拽排序（CE-03）、示例文案（CE-05）  
- 全局/模块高级样式（TS-03～TS-05）  
- 分页提示、预览缩放、双页模式（PV-02～PV-05）  
- 打印、图片、Markdown、外部导入、JSON 备份（IO-02～IO-06）  
- 自定义存储路径、历史版本、云同步（DS-02～DS-05）  
- 拼写/字数/布局优化、多语言 UI、引导教程（EF-01～EF-06）  
- 托盘、全局快捷键、窗口记忆、原生菜单、自动更新（DT-01～DT-06）  
- 在线链接、HTML、二维码、招聘平台格式（§6.9）

## 6. MVP 验收演示脚本（建议）

1. 启动应用（断网或可飞行模式） → 进入简历列表。  
2. **新建空白简历** → 进入编辑器；填写至少两个分区（如个人信息 + 一段工作经历）。  
3. 关闭某一模块显示 → 预览与导出中 **不出现** 该模块。  
4. 在 **≥2 套模板** 间切换 → 已填内容 **仍存在**。  
5. 触发 **自动保存**（等待防抖/定时）并 **Ctrl/Cmd+S** → 重启应用后数据仍在。  
6. 新建第二份简历并切换 → 未保存时得到 **保存/放弃/取消** 提示。  
7. **导出 PDF** → 用系统阅读器打开，版式与预览基本一致，中文正常。  

**辅助检查（本仓库）**

- 构建与静态断言：`pnpm run build && pnpm run verify:mvp`（不启动 GUI：校验 UTF-8、隐藏模块不出现在同源 HTML、三套模板 CSS 差异）。**不能**替代步骤 7 的 PDF 实机阅读。  
- 桌面包冒烟（当前 OS）：`pnpm run build && npx electron-builder --mac --dir`（macOS）；Windows / Linux 请在对应系统执行 `electron-builder --win` / `--linux` 或现有 `package.json` 脚本。
