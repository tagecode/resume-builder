# Resume Builder 产品需求文档（PRD）

| 属性 | 内容 |
|------|------|
| 产品名称 | Resume Builder（个人简历制作器，一款跨平台的桌面应用） |
| 文档版本 | 1.0 |
| 基线说明 | 与 `docs/feature.md` 功能清单一致；实现状态以本文 **§10 当前实现基线** 为准 |
| 技术栈 | Electron、Vite、React、TypeScript、Tailwind CSS v4、shadcn/ui、electron-store |

---

## 1. 背景与愿景

### 1.1 背景

求职者在多平台、多场景下需要维护**结构清晰、版式稳定、可快速迭代**的简历。纯在线工具存在隐私顾虑与弱网不可用问题；通用文档工具又缺乏简历专用的分区结构与预览体验。本产品以 **桌面端本地优先** 为定位，在离线环境下完成编辑、预览与导出。

### 1.2 产品愿景

打造一款**专业、可离线使用、数据归属用户**的跨平台简历制作软件：通过「分区编辑 + 实时预览 + 模板体系 + 可靠导出」降低排版成本，通过**本地持久化与可选备份**降低数据风险。

### 1.3 需求来源

- 功能范围以 `docs/feature.md` 为权威清单。
- 本 PRD 将其整理为可交付的用户故事、优先级、验收标准与非功能要求，并标注与现状的差距。

---

## 2. 目标与范围

### 2.1 产品目标

| ID | 目标 | 说明 |
|----|------|------|
| G1 | 本地可控 | 默认无需账号；简历与配置以用户磁盘为主存储 |
| G2 | 版式稳定 | 采用分区结构化编辑，避免自由画布带来的分页不可控 |
| G3 | 高效迭代 | 多简历管理、模板切换、自动保存、快捷导出 |
| G4 | 跨平台一致 | Windows / macOS / Linux 核心流程与数据格式一致 |

### 2.2 非目标（本期不做或仅预留扩展点）

- 不承诺内置招聘平台账号体系或一键投递（可作为远期对接）。
- 不将「云同步」作为 MVP 必选；以可选模块或后续版本规划。
- 在线托管简历链接若实现，**默认不强制依赖自有服务端**（可仅导出静态 HTML）。

## 3. 用户与场景

### 3.1 目标用户

- **应届生 / 实习生**：条目少、模板依赖高，需要引导与示例。
- **社会招聘人士**：版本多、需针对不同岗位微调多份简历。
- **重视隐私用户**：拒绝将完整简历长期托管在第三方云端。

### 3.2 核心场景

1. **从零创建**：新建空白或从模板创建 → 按区块填写 → 预览 → 导出 PDF。
2. **多版本维护**：复制简历 → 改名/排序 → 分别导出。
3. **应急修改**：打开最近编辑 → 自动恢复未保存草稿 → 导出/打印。
4. **备份迁移**：导出 JSON 备份 → 换机后导入恢复。

---

## 4. 术语与约定

| 术语 | 定义 |
|------|------|
| 简历项目 | 一份独立简历实例，含内容、模板与样式配置 |
| 区块 / 模块 | 简历内语义分区（如工作经历、技能） |
| 模板 | 控制整体版式与样式的预设；切换模板时内容结构映射到新版式 |
| 实时预览 | 右侧（或独立区域）根据内容与样式即时渲染版式预览 |

---

## 5. 信息架构（目标形态）

- **项目管理**：列表、搜索、排序、新建/复制/删除、最近打开。
- **编辑器**：分栏（编辑区 | 预览区）；左侧可为「大纲/模块树 + 表单」。
- **模板与样式**：模板库、全局样式（字体/字号/主题色/页边距/紧凑度）、模块级微调。
- **导入导出**：PDF/图片/打印/Markdown/JSON；数据备份与恢复。
- **设置**：存储路径、语言、拼写检查、快捷键说明、更新渠道（若启用）。

---

## 6. 功能需求（按模块）

优先级：**P0** 必须（MVP）、**P1** 重要、**P2** 增强、**P3** 可选/增值。验收标准采用可测试描述。

### 6.1 简历项目管理（对应 feature 一）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| PM-01 | 新建简历：空白 / 从模板创建 | P0 | 创建后进入编辑器；模板创建时预填示例结构可配置 |
| PM-02 | 简历列表：重命名、复制、删除 | P0 | 操作有确认与撤销入口（删除）；名称合法校验 |
| PM-03 | 列表：搜索、排序 | P1 | 按名称/更新时间等排序；搜索实时过滤 |
| PM-04 | 多简历切换 | P0 | 切换前处理未保存变更（提示保存/放弃/取消） |
| PM-05 | 自动保存与手动保存 | P0 | 默认定时或防抖自动保存；Ctrl/Cmd+S 手动保存有可感知反馈 |
| PM-06 | 最近编辑与草稿恢复 | P1 | 展示最近列表；异常退出后可恢复最近自动保存点 |

**数据与约束**：每份简历具有唯一 ID、显示名称、`updatedAt`；删除为软删除或二次确认由设计稿确定，需在 PRD 迭代中收口。

### 6.2 智能内容编辑（对应 feature 二）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| CE-01 | 分区编辑：个人信息、总结/意向、经历、教育、技能、项目、证书奖项、语言、自定义模块 | P0 | 各分区字段齐全；经历/项目等支持动态增减条目 |
| CE-02 | 轻量富文本 | P1 | 加粗、斜体、列表、链接等；粘贴时尽量净化无关 HTML |
| CE-03 | 区块顺序：拖拽排序 | P1 | 顺序持久化并在预览中同步 |
| CE-04 | 模块显示/隐藏 | P0 | 隐藏模块不出现在预览与导出中 |
| CE-05 | 智能提示 / 示例文案 | P2 | 空字段展示占位或「填入示例」操作 |

**内容模型建议（实现阶段落地）**：采用版本化的 JSON Schema 或 TypeScript 类型定义，便于导入导出与迁移。

### 6.3 模板与样式（对应 feature 三）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| TS-01 | 内置多套模板（行业/风格分类） | P0 | 至少 MVP 提供 2–3 套；分类可后续扩充 |
| TS-02 | 模板实时预览与切换 | P0 | 切换无丢字段；无法映射字段时明确提示或进入映射 UI |
| TS-03 | 全局样式：字体组合、字号/行高、主题色、页边距、紧凑度 | P1 | 变更即时反映预览；导出 PDF 与预览一致（允许字体嵌入策略说明） |
| TS-04 | 模块样式微调（时间线/卡片等） | P2 | 与所选模板能力绑定；无能力时灰显 |
| TS-05 | 用户自定义模板（保存当前样式为模板） | P2 | 可命名、列入「我的模板」 |

### 6.4 实时预览与页面适配（对应 feature 四）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| PV-01 | 分栏：左编辑右预览 | P0 | 布局可自适应中小屏（最小宽度遵循设计） |
| PV-02 | 预览模式：单页/双页滚动 | P2 | 模式切换状态记忆 |
| PV-03 | 页数与分页提示 | P1 | 溢出或临界分页时有提示或高亮 |
| PV-04 | 预览缩放 | P1 | 滑块或快捷键缩放 |
| PV-05 | 预览亮/暗主题（含打印意图模拟） | P2 | 与 UI 主题可区分；不影响最终 PDF 默认配置 |

### 6.5 导入与导出（对应 feature 五）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| IO-01 | 导出 PDF | P0 | A4/Letter、边距、质量可配置；中文与常用符号无明显乱码 |
| IO-02 | 导出图片 PNG/JPEG | P2 | 分辨率与背景可选 |
| IO-03 | 系统打印 | P1 | 调起系统打印对话框 |
| IO-04 | 纯文本 / Markdown 导出 | P2 | 结构可读；复杂排版以纯文本降级 |
| IO-05 | 导入外部简历 | P3 | JSON；LinkedIn 等格式为可选扩展，需版本说明 |
| IO-06 | 数据备份/恢复（JSON） | P1 | 导出完整可导入；校验版本不兼容时提示 |

### 6.6 本地数据与同步（对应 feature 六）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| DS-01 | 完全本地存储，首次无需注册 | P0 | 离线可用（除显式联网功能） |
| DS-02 | 自定义数据目录 | P2 | 设置中可选路径；迁移向导 |
| DS-03 | 自动备份与历史版本 | P2 | 保留可调份数/时间；支持回退 |
| DS-04 | 可选云同步（Gist/WebDAV/文件夹同步） | P3 | 安全说明与令牌存储方式文档化 |
| DS-05 | JSON Resume 标准导出 | P3 | 映射规则与覆盖率说明 |

### 6.7 辅助与效率（对应 feature 七）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| EF-01 | 拼写检查 | P2 | 依赖系统或集成库；可关 |
| EF-02 | 字数统计与长度建议 | P2 | 按区块或全文 |
| EF-03 | 一键优化布局 | P3 | 定义可量化规则（如 widow/orphan） |
| EF-04 | 快捷键 | P1 | 保存、导出等；提供帮助页 |
| EF-05 | 多语言界面 中文/English | P2 | 语言切换持久化 |
| EF-06 | 使用指南/引导 | P2 | 首次启动或帮助菜单进入 |

### 6.8 桌面端特色（对应 feature 八）

| ID | 需求 | 优先级 | 验收要点 |
|----|------|--------|----------|
| DT-01 | 系统托盘 | P2 | 显示/隐藏、快速打开最近 |
| DT-02 | 全局快捷键唤起 | P3 | 与系统冲突检测 |
| DT-03 | 窗口尺寸与位置记忆 | P1 | 多显示器场景合理恢复 |
| DT-04 | 多窗口 | P3 | 多实例或单实例多窗口策略需在技术方案中定稿 |
| DT-05 | 原生菜单（文件/编辑/视图/帮助） | P1 | 与快捷键一致 |
| DT-06 | 自动更新 | P2 | 渠道与安全；可推迟/跳过 |
| DT-07 | 离线可用 | P0 | MVP 不依赖网络 |
| DT-08 | 跨平台一致 | P0 | 三平台构建通过；路径与权限差异有说明 |

### 6.9 分享与发布（对应 feature 九）

| ID | 需求 | 优先级 | 验收要点 |

| SH-02 | 二维码 | P3 | 依赖 HTML 或 URL |
| SH-03 | 招聘平台格式对接 | P3 | 单独立项评估 |

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 自动保存防抖；大文档（多经历长描述）编辑仍保持可交互（目标帧率与上限在实现阶段 profiling） |
| 可靠性 | 写入失败可提示；避免半写坏文件（原子写或临时文件替换） |
| 安全 | `contextIsolation`、preload 白名单 API；不暴露任意 Node 至渲染进程 |
| 隐私 | 默认数据本地；若启用云同步，凭据存储方式（系统钥匙串/OS API）需设计 |
| 可访问性 | 核心按钮与表单可通过键盘操作；对比度遵循 WCAG 建议（分阶段） |
| 国际化 | 字符串外置；日期/数字格式随 locale |
| 兼容性 | 导出 PDF 在常用阅读器中显示正常；明确不支持的老系统范围 |

---

## 8. 数据与存储（概要）

- **简历正文**：建议单文件 JSON（或 SQLite 单库），含 `schemaVersion`、`resumeId`、`sections[]`、`style`、`templateId`。
- **应用设置**：主题、语言、路径、更新偏好等（现状已有 electron-store 存主题，可扩展）。
- **迁移**：`schemaVersion` 升级时提供只读升级或备份后再迁移。

## 9. MVP 与阶段规划（建议）

与 `docs/feature.md` 建议一致，并细化为发布节拍：

| 阶段 | 范围 | 对应优先级 |
|------|------|------------|
| **MVP** | 项目管理核心、分区编辑 P0 字段、至少 2 套模板、分栏实时预览、PDF 导出、本地持久化、手动/自动保存、模块显隐、离线 | P0 为主 |
| **V1.1** | 富文本、拖拽排序、打印、备份 JSON、窗口/菜单、分页提示 | P0–P1 |
| **V1.2** | 全局样式细化、图片导出、最近与草稿恢复、拼写与字数、自动更新 | P1–P2 |
| **后续** | 云同步、JSON Resume、自定义模板高级能力、分享增值 | P2–P3 |

## 10. 当前实现基线（截至文档编写时）

> 用于需求与实现的差距分析；随开发更新本小节或改为链接到 CHANGELOG。

## 11. 成功指标（可选量化）

| 指标 | 说明 |
|------|------|
| 任务完成率 | 用户能在首次启动 30 分钟内完成「新建 → 填写 → PDF 导出」 |
| 数据丢失率 | 异常退出后自动恢复成功率目标（实现备份策略后统计） |

## 12. 风险与依赖

| 风险 | 缓解 |
|------|------|
| PDF 版式与 Web 预览不一致 | 统一渲染管线或明确「打印/PDF 专用样式表」 |
| 富文本与分页预测困难 | MVP 可先结构化纯文本 + 轻量标记，再增强 |
| 跨平台路径与权限 | 数据目录、文件对话框统一抽象层 |
| 自动更新合规与签名 | macOS 公证/notarize、Windows 证书，纳入发布 checklist |

## 13. 开放问题（待决策）

1. 单实例还是多实例多窗口？与自动保存冲突策略。
